# 숙제 2: 나만의 쇼핑/마케 도우미 AI 챗봇

## 목표

특정 쇼핑몰 제품 페이지(예: 이니스프리)에서만 동작하는 **크롬 익스텐션** 플로팅 봇을 만들고,  
1) 리뷰를 최대 100개까지 수집한 뒤,  
2) 수집한 리뷰만 컨텍스트로 사용해 **Gemini 2.5 Flash Lite**로 질의응답합니다.

## URL 패턴

도메인 패턴 예시 (이니스프리 제품 상세):

- `https://www.innisfree.com/kr/ko/dp/product/103267`

위와 같은 패턴의 URL에 접속하면 오른쪽 하단에 플로팅 봇이 자동으로 나타나야 합니다.

## 시나리오

### 1단계: 리뷰 데이터 수집

1. 접속한 상품의 댓글(리뷰) 영역으로 이동.
2. **최신순** 정렬 선택 후, 정렬된 리뷰의 각 속성(작성일, 평점, 본문, 옵션 등)을 반복 수집.
3. 하단 **페이징**을 눌러가며 **최대 100개** 리뷰까지 수집.

### 2단계: AI에게 질의하기

1. 사용자가 입력한 질문을 리뷰 기반으로만 답변.
2. 리뷰 데이터가 많아 context에 모두 담기지 않을 경우를 대비해 **최적화** (요약/청크/임베딩 검색 등) 구성.
3. **LLM**: `gemini-2.5-flash-lite`
4. **API 키**: `.env`에서 로드 (예: `@gemini-api` 참고).

## 폴더 구성

- `extension/` — 크롬 익스텐션
  - `manifest.json` — Manifest V3, URL 패턴 `*innisfree.com/kr/ko/dp/product/*`
  - `content.js` — 제품 페이지 주입, 플로팅 버튼·패널 iframe
  - `content.css` — 플로팅 위치·스타일
  - `panel.html` / `panel.js` — 1단계 리뷰 수집 버튼·2단계 채팅 UI (Gemini 연동 시 백엔드 호출)
- `.env.example` — `GEMINI_API_KEY` 예시 (복사 후 `.env`에 키 설정)
- (선택) 백엔드/API — 리뷰 저장·Gemini 호출 (로컬 서버 또는 Cloud Function)
- `README.md` — 본 문서

## 쿠팡 식재료 특화 예시 프롬프트

쿠팡에서 식재료 등과 관련된 부분에 특화된 질문 예시:

- "이 상품 리뷰에서 **신선도·유통기한** 관련 언급만 요약해줘."
- "식재료 카테고리에서 리뷰에 자주 나오는 **키워드와 불만 사항**을 정리해줘."
- "같은 가격대 대비 **만족도가 높은 리뷰 문장 3개**만 추려줘."
- "배송·포장 상태 관련 리뷰만 골라서 요약해줘."

## 트립닷컴 호텔 (숙제 3와 연계)

호텔 리뷰가 수집된 경우 동일 챗봇 로직으로 확장 가능:

- "이 호텔 리뷰에서 **위치·청결도·가성비** 관련 의견만 요약해줘."
- "조식이나 주차 관련 불만이 있는 리뷰를 정리해줘."

## Linux 서버 / Windows 노트북 (SSH) 환경

| 환경 | 할 일 |
|------|--------|
| **Linux 서버** | 코드 보관·Git 동기화. 백엔드(Gemini API 호출)만 서버에서 실행 가능. |
| **Windows** | Chrome "압축해제된 확장 프로그램 로드"로 익스텐션 개발·디버깅. API 키는 Windows 로컬 서버 또는 서버 백엔드 URL로 연동. |

익스텐션은 **Windows(또는 Mac)의 Chrome**에서만 동작하므로, 실제 테스트는 Windows에서 진행하는 것을 권장합니다.
